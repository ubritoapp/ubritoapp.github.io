# Regras de Versionamento: Portfólio

O Antigravity deve seguir estas regras ao gerenciar este repositório:

1. **Mensagens de Commit:** Use o padrão simplificado:
   - `feat:` para novas seções ou projetos adicionados.
   - `fix:` para correção de bugs visuais ou links quebrados.
   - `docs:` para alterações em textos ou README.
2. **Automação de Commit:** - Sempre revise o `git diff` antes de commitar.
   - Informe ao usuário quais arquivos estão sendo alterados antes de concluir.
3. **Impedimentos:**
   - Não realize commit se houver chaves de API ou emails expostos no código.
   - Mantenha o histórico limpo e atômico (um commit por alteração).
