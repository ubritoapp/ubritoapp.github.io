1. Visão Geral
Objetivo: Criar uma landing page de portfólio profissional e minimalista.

Hospedagem: GitHub Pages (username.github.io).

Stack: HTML5, CSS3 (Mobile-first) e Vanilla JavaScript (se necessário).

2. Estrutura de Pastas (Arquitetura)
Plaintext
/
├── .antigravity/       # Regras e logs de automação
├── assets/             # Imagens e ícones
├── css/
│   └── style.css       # Estilização modular
└── index.html          # Ponto de entrada único
3. Requisitos Funcionais (Seções)
3.1 Header (Cabeçalho)
Nome do profissional em destaque (H1).

Menu de navegação fixo (Home, Projetos, Sobre, Contato).

Link para o perfil do LinkedIn ou GitHub.

3.2 Seção Hero
Breve descrição: "Arquiteto de Soluções & Especialista em Segurança".

Chamada para ação (CTA): Botão "Ver Projetos".

3.3 Grid de Projetos (Placeholders)
Layout: Grid responsivo (1 coluna mobile, 3 colunas desktop).

Conteúdo: 6 caixas (cards) de placeholder.

Estilo do Card: * Borda suave ou sombra leve.

Fundo cinza claro (#f4f4f4).

Texto centralizado: "Projeto em Desenvolvimento...".

Efeito hover: Leve levantamento ou mudança de cor.

3.4 Rodapé
Copyright com ano dinâmico.

Links de redes sociais.

4. Requisitos Não Funcionais (Segurança & Performance)
Shift-Left Security: Nenhum dado sensível (chaves, senhas) no código.

Performance: Uso de fontes seguras do sistema (inter, system-ui) para evitar carregamento externo pesado.

Responsividade: O layout deve se adaptar a telas de 320px até 1920px.
